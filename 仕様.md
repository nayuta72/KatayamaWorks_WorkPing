# WorkPinプロジェクト
WorkPinという名前のツールを開発します。

これは部署内のメンバーが全員使うツールです。
これは出退勤時間とそのときの体調をメンバー間で共有するためのツールです。

## メインウィンドウ
- ツールは**NavigationView**が配置されていて、2ページあります。
- 右上の最小化ボタンの左には**ユーザアカウント設定画面**(モーダル)を起動するためのボタンがあります。
- ユーザアカウント設定画面起動のボタンの左には、コンボボックスがあり、どのログファイルに書き込むかを切り替えることができます
- ツール起動時にはsettings.json(ユーザアカウント設定画面で保存)やフォルダが存在するかを確認し、なければ作成するようにしておく
- ツール起動時にsettings.jsonを読み込み、情報を保持しておく。ツールのどこからでもアクセスできるようにする
- ツール起動時にsettings.jsonを読み込んだときに、姓、名、ユーザID、XMLファイルのパス少なくとも一つが設定されていない場合は、メッセージダイアログを出して、ユーザにアカウントを設定するように促す。
- ツール起動開始直後から定期的(5分毎)にsettings.json内に入力されている出退勤パスがアクセス可能かを確認する。アクセス不可能な状態の場合は、画面タイトルの右側にアクセスができていないことを示すアイコンを表示させる。
- settings.json内に出退勤パスが入力されているかつそのファイルパスが有効な場合は、ツール起動中、常に監視しておき、ファイルの変更があった場合には、Window11の画面右下にLastLogの内容を通知(名前と顔アイコン(色付き))として表示させる。(管理者のみが有効な機能)
-ツール起動時から

## ユーザアカウント設定画面
ユーザの各種情報や読み書きするファイルやフォルダのパスを、設定ファイルに書き込む処理をする画面です。

### 配置するコントロール
- テキストボックスを横に3つ：ユーザ名(姓と名)、ユーザID入力する
- テキストボックス：部署名を入力する
- チェックボックス：管理者かどうかを設定する
- テキストボックス：出退勤ログを書き込むXMLファイルのパスを入力する。出退勤ログファイルのパスは3つまで設定できるようにする。また、パスにはどこの部署用かの名前を入れるようにする。部署を兼務している人ように。
- コンボボックス：デフォルトのログファイルパスをどれにするかを指定する。
- ボタン：画面の情報を保存する

### 動作
- ユーザアカウントの情報はRoamingフォルダの中にあるkikakutoolsフォルダの中のWorkPinフォルダに入っているsettings.jsonというファイルに書き込む
- 保存ボタンを押したら姓、名、管理者かどうか、出退勤ファイルのパスをjson形式で書き込む
- ユーザアカウント画面を起動時にはsetting.jsonを読み込んで画面内に情報をセットしておく


## NavigationView 1ページ目
出退勤の通知やパソコンのログの情報を表示する画面です。

### 配置するコントロール
- テキストブロックを縦に2つ：昨日のWindowsへのログイン時刻とログアウト時刻を表示する
	1ページ目が表示された直後にWindowsのシステムの情報から昨日の一番最初のログイン時刻と一番最後のログアウト時刻を取得し、テキストブロックにセットします。
- テキストブロックを縦に2つ：今日のWindowsへのログイン時刻と現在の時刻を表示する
	1ページ目が表示された直後にWindowsのシステムの情報から今日の一番最初のログイン時刻と現在時刻(リアルタイムに変化)を取得し、テキストブロックにセットします。
- トグルスイッチ：「出社」か「在宅」かを切り替えます。
- ボタンを横に3つ：出社時の状態を通知するためのボタン。調子がよさそうな笑顔アイコンボタン、いつもどおりの真顔な顔アイコンボタン、体調悪そうな顔アイコンボタンを用意する
	ボタンを押すと、Setting.jsonと出退勤ログファイルに、出社時の状態として名前とステータスと時刻を書き込む。また、押されたボタンの色をステータスにあわせて緑(良い)、黄色(普通)、赤(悪い)に変える。そのタイミングで出退勤ログファイルにアクセスできないときは、定期的にアクセスできるか確認しているタイミングでアクセスできるようになったときに書き込みに行く。
- ボタンを横に3つ：退勤時の状態を通知するためのボタン。順調に仕事が終わった笑顔アイコンボタン、少しつかれたけど計画どおり終えた真顔な顔アイコンボタン、負荷が高すぎて大変そうな顔アイコンボタンを用意する
	ボタンを押すと、Setting.jsonと出退勤ログファイルに、退勤時の状態として名前とステータスと時刻を書き込む。また、押されたボタンの色をステータスにあわせて緑(良い)、黄色(普通)、赤(悪い)に変える。そのタイミングで出退勤ログファイルにアクセスできないときは、定期的にアクセスできるか確認しているタイミングでアクセスできるようになったときに書き込みに行く。
	
- ボタン：コメントを書き込むコンテンツダイアログを表示させる。

### 動作
- XMLに出退勤のログを書き込む際には、Rootの直下の一番上に今日の日付から取得した文字列(yyyyMMdd)_ユーザIDでエレメントを作成し、そこに名前、出社か在宅か、出社時刻、出社時のステータス、退社時刻、退社時のステータスを属性として入力する。すでに同じエレメントが存在する場合は新規作成する必要なしで、もとあったエレメントに上書きする。ステータスは◎、◯、✕にする。
- コメント入力ダイアログでコメント(文字数制限：10文字まで)を入力し、「送信」ボタンを入力すると、出退勤のエレメントに用意されているComment属性に書き込む
- エレメント/属性の書き込みと同時にRoot直下のエレメントの一番最後に**LastLog**という名前のエレメントも作成し、そこに今回書き込んだ属性の情報を上書きしておく。(書き込んでない属性値はリセット)
- 自分が今日押したステータスの状態や時刻はSetting.json内に保持しておいて、ツールを再起動したときにも色を再セットできるようにしておく。
