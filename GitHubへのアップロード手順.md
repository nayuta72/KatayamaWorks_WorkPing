# VSCode & Claude Code プログラム開発・管理マニュアル

このドキュメントでは、VSCodeとClaude Codeを用いたプログラム開発において、GitHubへ正しくアップロードする手順と、複数プロジェクトを混同しないための管理手法についてまとめています。

---

## 1. GitHubへのアップロード手順

### 【初回】リポジトリの作成と初期設定
1. **GitHub側での準備**
   - GitHubで [New repository] を作成。
   - READMEや.gitignoreの自動生成チェックは**すべて外す**。
   - 作成後に表示される **HTTPSのリポジトリURL** をコピー。
2. **VSCode側での操作**
   - プロジェクトフォルダを開く。
   - 左側の「ソース管理」アイコンから **[リポジトリを初期化する]** をクリック。
   - メッセージを入力して **[コミット]**。
   - **[ブランチの公開]** を押し、コピーしたURLを紐付けて送信。

### 【2回目以降】日常的なコミット & プッシュ
1. **変更の確認**: 「ソース管理」パネルで修正内容を確認。
2. **ステージング**: ファイル横の **[+]** ボタンを押し、「ステージ上の変更」に移動。
3. **コミット**: 変更内容の説明を入力して **[コミット]**。
4. **プッシュ**: 左下の **同期アイコン（くるくるマーク）** をクリックしてGitHubへ反映。

## 2. 複数製品（MWStudio / WorkPin）の混同防止策

2つの製品を並行開発する際、アップロード先を間違えないための対策です。

### 視覚的な区別
- **VSCode拡張機能「Peacock」の利用**:
  プロジェクトごとにVSCodeのウィンドウ枠の色を変える（例：MWStudioは青、WorkPinは緑）。視覚的にどちらを操作しているか一瞬で判断できます。
- **ワークスペースの分離**:
  「1つのプロジェクトにつき、1つのVSCodeウィンドウ」を徹底する。1つの画面に2つのフォルダを混ぜる「マルチルート・ワークスペース」は誤操作の元になるため避ける。

### Git設定の確認
- **リモートURLの点検**:
  ターミナルで `git remote -v` を実行し、現在開いているフォルダが正しいGitHubリポジトリと紐付いているか確認する。

## 3. Claude Code 使用時の注意点

Claude Codeに対して「コミットしてプッシュして」と指示する場合の安全性について。

### 基本的な安全性
- **動作の仕組み**: Gitの設定は各プロジェクトフォルダ内の `.git` に保存されています。Claude Codeは実行しているカレントディレクトリの設定を使用するため、**MWStudioのフォルダで実行している限り、勝手にWorkPinへ送られることはありません。**

### より安全な運用ルール
1. **コミットまでを任せる**: 
   Claude Codeには「変更内容をコミットして」と指示するに留め、最後の「プッシュ（GitHubへの送信）」はVSCodeのボタンを自分で押すようにすると、送信直前に目視確認ができて確実です。
2. **ターミナルのパス確認**: 
   Claude Codeを起動する際、ターミナルのカレントディレクトリが現在作業したいプロジェクト（MWStudio or WorkPin）になっているかを必ず確認する。
3. **親子フォルダの禁止**: 
   プロジェクトを親子関係のフォルダにしない。必ず独立したディレクトリ（兄弟関係）として管理する。
